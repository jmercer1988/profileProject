About This Component Library

This is a collection of Angular components and services designed for learning and practicing key concepts:

Parent-Child Component Relationships

The library emphasizes the use of parent and child components to manage data flow and create modular UIs.

  • Parent Components: Responsible for fetching data, handling logic, and passing data down to child components.
  • Child Components: Focus on presenting the data and emitting events back to the parent component.

Dumb (Presentational) Components

Many of the components in this library are "dumb" or presentational components. This means:

  • They primarily focus on displaying data received through inputs ('Input').
  • They notify their parent of user interactions through events ('Output').
  • They have minimal internal logic and don't directly depend on external services.

Examples

Explore the different components in this library to see these concepts in action. You'll find examples of:

  • Headers and footers with customizable links.
  • Cards for displaying information.
  • Forms with two-way data binding.
  • Toast notifications for user feedback.
  • ...and more!
